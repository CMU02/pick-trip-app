# 핵심 기능

## 1. 로그인 / 회원가입

> 일정 저장·공유 기능을 사용하기 위해 로그인이 필요하다. 인증 흐름은 Spring 백엔드 서버가 담당한다.

- Kakao 소셜 로그인
- Google 소셜 로그인
- 비로그인 상태에서도 컨텐츠 탐색은 가능

## 2. 지역 컨텐츠 큐레이션

> 하동, 영주, 예천의 음식, 축제, 관광지, 자연, 문화, 체험 컨텐츠를 카드 형태로 제공합니다. <br>
> 각 컨텐츠는 단순 설명이 아니라 여행 판단에 필요한 정보를 함께 제공해야합니다.

### 검색

- 컨텐츠명, 장소명, 카테고리 기반 검색 가능
- 지역 선택 후 검색 범위를 해당 지역으로 제한

### 컨텐츠 카드에 포함될 정보

- 장소명 또는 콘텐츠명
- 카테고리
- 한 줄 소개
- 추천 대상
- 예상 소요시간
- 운영시간
- 휴무일
- 위치
- 주차 가능 여부
- 아이 동반 적합도
- 부모님 동반 적합도
- 실내·실외 여부
- 계절성
- 예약 필요 여부
- 축제의 경우 개최 기간

## 3. 여행 바구니

> 사용자가 마음에 드는 콘텐츠를 담아 일정 생성의 재료로 사용하는 기능입니다. <br>
> 여행 바구니는 단순 저장 목록이 아니라 AI 일정 생성의 입력값 역할을 한다. 사용자는 각 콘텐츠의 우선순위를 설정할 수 있다.

- `꼭 가기`: 일정에 반드시 포함
- `가면 좋음`: 동선과 시간이 맞으면 포함
- `시간 남으면`: 여유 시간이 있을 때 후보로 사용

## 4. AI 일정 생성

> AI는 사용자가 선택한 컨텐츠와 여행 조건을 바탕으로 일정을 생성합니다.

### AI가 고려해야 할 요소

- 여행 날짜와 기간
- 장소별 운영시간
- 축제 개최 기간
- 식사 시간
- 장소별 예상 체류 시간
- 이동 거리와 이동 순서
- 동행 유형
- 걷기 부담
- 실내·실외 비율
- 날씨 또는 우천 대안
- 사용자가 설정한 우선순위

> AI 일정 결과에는 판단 이유를 함께 제공해야한다.

### Example

- "축제 운영시간이 오전 10시부터 라서 1일차 오전에 배치했습니다."
- "아이와 함께 이동하기 쉽도록 가까운 장소끼리 묶었습니다."
- "점심 시간대에 지역 대표 음식을 먹을 수 있도록 식당을 먼저 배치했습니다."

## 5. 일정 수정 및 재생성

> 사용자는 AI가 만든 일정을 그대로 따르지 않고 직접 수정할 수 있어야 한다.

- 장소 순서 변경
- 장소 삭제
- 대체 장소 추가
- 특정 장소 고정
- 전체 일정 다시 생성
- 하루 단위로 다시 생성

## 6. 가족 여행 편의 필터

> 가족 여행자의 의사결정을 돕기 위해 일반 여행 앱보다 현실적인 조건 필터를 강화한다.

### 필터 예시

- 아이와 가기 좋음
- 부모님과 가기 좋음
- 걷기 적음
- 주차 편함
- 실내 가능
- 비 오는 날 가능
- 화장실 이용 편함
- 예약 필요
- 반려동물 동반 가능 여부
