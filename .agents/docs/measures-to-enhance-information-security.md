# 정보 신뢰성 보완 방안

AI 일정 생성의 품질은 데이터 신뢰도에 크게 의존한다.
따라서 서비스 초기에는 완전 자동 수집보다는 검수된 데이터를 중심으로 운영해야한다.

- 한국관광공사 오픈 API(TourAPI)를 1차 데이터 소스로 사용
- 지자체 공식 축제 정보를 보조 데이터로 활용
- 운영시간, 휴무, 축제 기간, 예약 여부는 별도 필드로 구조화
- 업데이트 날짜를 콘텐츠 카드에 표시
- 불확실한 정보는 "방문 전 확인 필요"로 표시
- 사용자가 잘못된 정보를 제보할 수 있는 기능 제공
- AI가 생성한 일정에는 정보 출처와 생성 기준을 함께 제공

## 데이터 갱신 주기

컨텐츠 유형별로 갱신 주기를 다르게 적용한다.

| 데이터 유형                        | 갱신 주기                          |
| ---------------------------------- | ---------------------------------- |
| 기본 정보 (장소명, 위치, 주차 등)  | 주 1회                             |
| 운영시간 · 휴무일                  | 일 1회                             |
| 축제 · 이벤트                      | 축제 시작 1주 전 ~ 종료일까지 일 1회 |
