# Key Usage Flow

1. 로그인 / 회원가입

- Kakao 또는 Google 소셜 로그인
- 인증 흐름은 Spring 백엔드 서버가 담당
- 비로그인 상태에서도 컨텐츠 탐색은 가능하나, 일정 저장·공유는 로그인 필요

2. 사용자가 여행 지역을 선택

- 하동
- 영주
- 예천

3. 여행 날짜와 기간을 선택

- 당일치기
- 1박 2일
- 2박 3일
- 커스텀

4. 동행 조건을 선택

- 아이와 함께
- 부모님과 함께
- 가족 전체
- 걷기 적게
- 자연 위주
- 체험 위주
- 음식 위주
- 실내 대안 필요

5. 지역 컨텐츠를 탐색

- 음식
- 축제 및 행사
- 자연 관광지
- 문화 및 역사
- 체험
- 시장 및 로컬 상권
- 비 오는 날 갈 곳

6. 사용자가 원하는 컨텐츠를 담는다.

- `모바일`에서는 `담기` 버튼 중심
- 선택한 컨텐츠는 `꼭 가기`, `가면 좋음`, `시간 남으면`으로 우선순위 설정 가능

7. 완료 버튼을 누르면 AI가 일정 생성을 한다.
8. 사용자는 생성된 일정을 확인하고 수정한다.

- 장소 순서 변경
- 장소 삭제
- 대체 장소 추가
- 식사 시간 변경
- 일정 다시 생성

9. 최종 일정을 저장하거나 공유한다.
